## **1. Project Overview**

**Japan Trip Planner** is a small web application designed to demonstrate a complete Git-based workflow while integrating generative AI into the development process.  
The project focuses on:

- Building a functional and visually clear web interface  
- Using AI as a development assistant  
- Applying a full Git workflow exclusively through the command line  
- Publishing the final result using GitHub Pages  

This README documents both the technical aspects of the project and the workflow followed during development.

---

## **2. Project Description**

The application allows users to explore an illustrated map of Japan and access itineraries for several major cities.  
The interface is simple, intuitive, and fully implemented with **HTML**, **CSS**, and **JavaScript**.

### **Main Features**
- **Interactive map** of Japan  
- **City buttons**: Tokyo, Kyoto, Osaka, Hiroshima, and Nara  
- **Dynamic itinerary display** using JavaScript  
- **Automatic content update** when selecting a city  

### **Each itinerary includes**
- City overview  
- Main attractions  
- Local cuisine  
- Transportation tips  
- Best time to visit  

The goal is not to create a full travel platform, but to demonstrate clean structure, dynamic content handling, and proper version control.

## **5. Git Workflow Summary**

The project was developed following a structured Git workflow, executed entirely through the command line.

### **Workflow Steps**

| **Step** | **Description** |
|----------|------------------|
| **Repository Initialization** | Created the project repository using `git init`. |
| **Tracking Changes** | Added files to staging with `git add` and verified status with `git status`. |
| **Committing** | Created the initial commit and subsequent updates. |
| **Branching** | Created the `feature-update` branch to add the Nara itinerary. |
| **Merging** | Merged the feature branch back into `main` using `git merge`. |
| **Remote Setup** | Connected the local repository to GitHub and pushed changes. |
| **Deployment** | Published the project using GitHub Pages. |

This workflow ensures clean development, organized version history, and safe integration of new features.

---

## **6. Deployment**

The final version of the project was deployed using **GitHub Pages**, allowing the application to be accessed through a public URL.  
Deployment steps included:

1. Pushing the repository to GitHub  
2. Enabling GitHub Pages in the repository settings  
3. Selecting the `main` branch as the source  
4. Verifying the site once published
